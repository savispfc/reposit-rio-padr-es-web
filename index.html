Respostas às Perguntas

a) Atributo "placeholder" em <input type="text">: O atributo "placeholder" é usado para fornecer um texto de exemplo ou uma dica dentro de um campo de entrada (input) do tipo "text". Ele fornece uma breve descrição ou instrução sobre o que o usuário deve inserir nesse campo. O texto de exemplo desaparece quando o usuário começa a digitar no campo.

b) Atributo "checked" em <input type="radio"> ou <input type="checkbox">: O atributo "checked" é usado para indicar que um botão de rádio (radio button) ou uma caixa de seleção (checkbox) deve estar marcado inicialmente quando a página é carregada. Se "checked" estiver presente no elemento, o botão de rádio ou a caixa de seleção estará marcado por padrão.

c) Atributo "value" em <input>: O atributo "value" é usado para definir o valor inicial ou o valor que será enviado ao servidor quando um formulário for enviado. Em campos de entrada de texto, "value" define o texto que aparece no campo por padrão. Em elementos de botão, "value" define o valor que será enviado ao servidor quando o botão for clicado.

d) Atributo "name" para agrupar botões de rádio: Para agrupar vários botões de rádio de modo que apenas um deles fique marcado de cada vez, você deve usar o atributo "name" com o mesmo valor para todos os botões de rádio que pertencem ao mesmo grupo. Isso garante que apenas um deles possa ser selecionado ao mesmo tempo.

e) Tag <label> e atributo "for": A tag <label> é usada para criar uma rótulo associado a um campo de entrada (input), como um campo de texto ou uma caixa de seleção. O atributo "for" em <label> especifica qual campo de entrada ele está associado, usando o valor do atributo "id" desse campo de entrada. Isso melhora a acessibilidade e permite que os usuários cliquem no rótulo para focar ou selecionar o campo de entrada.

f) Atributo "selected" em <option>: O atributo "selected" em <option> é usado para indicar que uma opção em um menu suspenso (select) deve ser selecionada por padrão quando a página é carregada.

g) Atributo "required" em tags de formulário: O atributo "required" é usado para indicar que um campo de entrada em um formulário é obrigatório. Isso significa que o usuário deve preencher esse campo antes de enviar o formulário.

h) Tag <textarea>: A tag <textarea> é usada para criar um campo de entrada de texto multilinha em um formulário, permitindo que os usuários insiram texto mais longo, como comentários ou mensagens.

i) Atributo "action" em <form>: O atributo "action" em <form> especifica para onde os dados do formulário devem ser enviados após o envio. Ele contém o URL do servidor que processará os dados do formulário.

j) Atributo "method" em <form>: O atributo "method" em <form> especifica como os dados do formulário serão enviados ao servidor. Os valores comuns são "GET" (anexa os dados à URL) e "POST" (envia os dados no corpo da solicitação HTTP). Para ocultar o valor digitado da senha na barra de endereços do navegador, você deve usar "POST".

k) Para que o valor digitado de uma senha não apareça na barra de endereços do navegador, você deve usar o atributo "method" com o valor "POST" na tag <form>. Usar o método "POST" garante que os dados do formulário, incluindo a senha, sejam enviados no corpo da solicitação HTTP, em vez de serem anexados à URL como acontece com o método "GET". Dessa forma, os dados sensíveis, como senhas, não serão visíveis na barra de endereços do navegador.

l) Atributo "enctype" para upload de arquivos: Para formulários que serão usados para fazer upload de arquivos, você deve definir o atributo "enctype" em <form> como "multipart/form-data". Isso permite que os arquivos sejam enviados com sucesso ao servidor.

m) Tags <fieldset> e <legend>: A tag <fieldset> é usada para agrupar elementos relacionados em um formulário e criar uma caixa de borda ao redor deles. A tag <legend> é usada dentro de <fieldset> para fornecer um rótulo ou legenda para o grupo de elementos relacionados. Isso ajuda a organizar e rotular partes específicas de um formulário.
